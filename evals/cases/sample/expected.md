# Atomic notes capture one idea per note

## Core Idea
Each permanent note should contain exactly one idea, stated as a claim rather than a topic. Structure emerges from links between atomic notes, not from folder hierarchies.

## Elaboration
When notes are atomic, you can link them precisely. A note about "Observer pattern" can link to "Pub-Sub pattern" without also dragging in unrelated content about "event-driven architecture" that happened to be in the same file.
